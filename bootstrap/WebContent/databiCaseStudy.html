1.4 Case Study : COTS-Anti Money Laundering
Environment: Oracle9i, Business Objects 5.1, Infoview, Broad cast agent, TWS & Perl Scripts.

COTS-Anti-Money Laundering system allows suspicious customer activities to be detected throughout customer relationship and the data from COTS, FLEXCUBE systems in Oracle database and populate into AML database by using stored procedures, Db link in TWS scheduler jobs and generate the analysis reports for Low risk, High risk, Watchlist, Account setup, Sign-Off and Audit reports in various levels i.e. as per Country, Branch, Cost centre and Customer by using Business Object Universe, Business Object report module & reports scheduling by using BCA (Broad case agent – Business Object tool) and send to InfoView users(View BO reports in INFOVIEW) and PERL Script used to format Business objects reports in text format (ROD format i.e. including banner).

The roles and responsibilities of Optimum consultants were:

  • Analysis and design of COTS-Anti-Money Laundering system.
  • Design of database, working tables, Summary tables, reference tables & history tables using SQL * Loader and PL/SQL programming ((stored procedures, functions, packages & triggers) with Shell scripts in UNIX environment.
  • Design of Oracle forms for setup Parameters & Accounts.
  • Design of Universe by using Business Object Designer like reusable to all countries and generates the reports by using Business Object report module and scheduled by Broad cast agent (This tool used to schedule, refresh & send business documents to InfoView users) and PERL Script to format Business Objects reports in ROD text format.
  • Involved in SIT, UAT & Production support.
  • Implementation of project & Cutover and preparation of Survival, Operational and Technical documents including User training.

1.6 Case Study : Management Information System for Trade Finance
Environment: Oracle8i, Cognos, Oracle Discoverer and Oracle Express.

The group is responsible to develop and commission a management information system to provide a globalize view of the Trade finance (TF) related activities of Deutsche Bank. Currently the project TFMIS is catering to the MIS needs of TF for projecting Revenues & costs involved in TF transactions resulting in getting customer and product profitability, Outstanding amount, turnover etc. We have developed a DataMart for storing the currently and historical transactions.

Optimum consultants have been involved in STAR schema design for storing the huge volume of data, approximately 500 GB in oracle database using oracle tools. Additionally to perform the extraction and transformation few modules have been built which continuously poll the TF production systems and gather / filter / process data for MIS purposes.

Discoverer & Express (Oracle tools) are used to perform BI analysis. Currently this application is being ported onto Web using Cognos BI tools, which is expected to cater to the needs of larger user base.
